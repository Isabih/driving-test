# Local Field Server Setup

This document explains the recommended on-site deployment for cameras, restreaming, active-stage processing, and communication with the main backend.

## Recommended local machine

Preferred:
- Intel mini PC / Dell OptiPlex / NUC class machine
- 16 GB RAM minimum, 32 GB preferred
- 1 TB SSD
- Ubuntu Server 22.04 or 24.04
- Docker + Docker Compose

## Local responsibilities

The local field server handles:
- ingesting ONVIF / RTSP camera feeds
- restreaming camera feeds to browser-friendly outputs
- running the stage-aware vision service
- optionally serving as the local MQTT hub
- forwarding snapshots and events to the main backend

## Install base packages

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg lsb-release git ffmpeg python3 python3-venv python3-pip
```

## Install Docker

```bash
sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
sudo usermod -aG docker $USER
```

Re-login after adding your user to the docker group.

## Suggested folders

```bash
sudo mkdir -p /opt/apaforme/{media,postgres,mosquitto,backend,vision,evidence,firmware}
sudo chown -R $USER:$USER /opt/apaforme
```

## Start the local stack

```bash
cd deployments/local-server
docker compose up -d --build
```

## Services in the local stack

- `db` → PostgreSQL
- `mosquitto` → local MQTT broker
- `mediamtx` → camera restreaming for RTSP/WebRTC/HLS
- `backend` → FastAPI API
- `vision_service` → active-stage-only inference controller skeleton

## How stage-aware camera processing works

The vision service uses the active stage from the exam backend and opens only those camera streams.

Example:
- `figure_8` → `cam_s1_overhead`, `cam_s1_side_a`
- `zigzag` → `cam_s2_main`
- `narrow` → `cam_s3_main`
- `emergency_stop` → `cam_s4_main`

When stage transitions happen, the backend changes `current_stage_key`, and the vision service updates the active stream set.

## Browser live streaming

The dashboard should consume `viewer_url` values from the camera stage config. These can point to:
- WebRTC stream URL from MediaMTX
- HLS URL from MediaMTX
- stream link generated by NVR/VMS restreaming

## Production notes

- Put all field cameras on a dedicated PoE switch / VLAN.
- Use static DHCP reservations or static IPs for cameras.
- Keep the local field server on wired Ethernet.
- Upload only events + snapshots to the cloud if internet is unstable.
